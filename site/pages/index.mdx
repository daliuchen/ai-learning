## 写作约定

所有手册都遵循同一套约定：

- **每篇结构统一**：一句话总结 → 概念 → 最小代码 → 进阶 → 生产建议 → 常见坑（表格）→ demo 入口
- **代码全部可独立运行**：每段 import 完整，不依赖前文
- **关键 API 写完整路径**：例如 `pydantic_ai.Agent` 而非只写 `Agent`
- **错误示范有标注**：`# ❌ 错误写法` / `# ✅ 正确写法` 对比
- **横向对比**：合适的地方插入相邻框架（如 PydanticAI vs LangChain）的等价代码
