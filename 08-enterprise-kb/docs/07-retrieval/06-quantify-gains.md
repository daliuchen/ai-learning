# EKB 37：量化提升——用数据复盘每一项增强

> **一句话**：检索增强加完了，回过头用一张表把每项的贡献摊开——这正是「评估先行」最大的回报。它让你知道每个增强**真实值多少**，哪些该留、哪些可砍、哪些其实没用。这种「数据驱动复盘」是教程从不展示、却最有说服力的东西。

---

## 1. 完整的提升账本

从 MVP baseline 到全套增强，每一步的数字（示意，实际以你的评估集为准）：

| 阶段 | recall@5 | recall@3 | 引用准确率 | 延迟 | Δrecall@5 |
|------|----------|----------|-----------|------|-----------|
| MVP 单路向量 | 0.66 | 0.58 | 0.70 | 120ms | — |
| + BM25 混合 | 0.79 | 0.68 | 0.74 | 160ms | **+0.13** |
| + Rerank | 0.81 | 0.83 | 0.86 | 320ms | +0.02 |
| + Query 改写 | 0.86 | 0.85 | 0.88 | 600ms | **+0.05** |

一眼读出的结论：
- **混合检索**贡献最大的 recall 提升（+0.13）——专名/口语错配确实是主要痛点
- **rerank** 对 recall@5 提升不大（+0.02），但**对 recall@3 和引用准确率提升巨大**（头部质量）
- **query 改写**再补 +0.05，但延迟翻倍——值不值要看场景

---

## 2. 这张表回答了三个关键问题

### 问题一：每个增强值多少？

不再是「感觉加了 rerank 更好」，而是「rerank 让 recall@3 从 0.68 到 0.83」。**有数字，决策才有底气。**

### 问题二：哪些可以砍？

如果某项增强 Δ 接近 0，它就是**纯成本**，该砍。比如假设 query 改写只带来 +0.01 却让延迟翻倍——那这个场景就不该上改写。数据让你敢砍。

### 问题三：边际收益在哪里递减？

从表里看，前两项（混合）收益大，后面递减。这告诉你**优化的精力该投在哪**——继续抠 rerank 参数，不如回去看看分块质量或补数据。

---

## 3. 不要只看平均分，看分布

平均 recall 涨了，不代表每类问题都涨。按类型拆开看：

```
              直球类   口语类   专名类   跨文档
MVP           0.85     0.55     0.50     0.45
全套增强       0.92     0.84     0.85     0.70
```

跨文档类（答案散在多篇）即使全套增强也只到 0.70——这暴露了**下一个该攻的方向**（可能要调分块或增加召回数）。**平均分会掩盖薄弱环节，分类型看才能发现真问题。**

---

## 4. 复盘也要看「红线」和「兜底」

检索增强主要冲 recall，但别忘了同时盯住企业知识库的红线指标：

```
              兜底正确率   越权召回
MVP           0.90        0（MVP还没接权限，全部all）
全套增强       0.95        0
```

注意：rerank 阈值收紧了兜底（[35 篇](./04-rerank.md)），兜底正确率从 0.90 升到 0.95——**检索增强顺带改善了兜底**。越权召回此时还是 0，因为权限要等第 08 章才真正接入并验证。

---

## 5. 把复盘沉淀成「实验记录」

每次增强的评估结果，建议存成一个简单的实验日志，而不是跑完就忘：

```jsonl
{"exp": "baseline", "recall5": 0.66, "recall3": 0.58, "cite": 0.70, "note": "单路向量"}
{"exp": "hybrid",   "recall5": 0.79, "recall3": 0.68, "cite": 0.74, "note": "+RRF混合"}
{"exp": "rerank",   "recall5": 0.81, "recall3": 0.83, "cite": 0.86, "note": "+bge-reranker"}
{"exp": "rewrite",  "recall5": 0.86, "recall3": 0.85, "cite": 0.88, "note": "+查询改写,延迟翻倍"}
```

这份记录的价值：半年后有人问「为什么用混合检索不用纯向量」，你有数据；想回退某个增强时，知道会损失多少。**实验记录是项目的记忆。**

---

## 6. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 不复盘，加完就完 | 不知每项价值 | 做提升账本 |
| 只看平均分 | 薄弱类型被掩盖 | 分类型看分布 |
| 收益≈0 的增强也留着 | 纯成本/延迟 | 数据说话，敢砍 |
| 只盯 recall 忘了红线 | 兜底/越权退化没察觉 | 同时盯红线指标 |
| 实验结果不记录 | 决策无据、回退无凭 | 沉淀实验日志 |
