# EKB 33：BM25 关键词检索——补向量检索的短板

> **一句话**：向量检索擅长「语义相近」，但对**专有名词、缩写、精确术语**经常失手——因为 embedding 把「OA」「VPN」「SKU-2024」这类罕见词的语义压糊了。BM25 这种关键词检索恰好相反，精确匹配字面词。两者互补，所以要一起用。本篇加上 BM25。

---

## 1. 向量检索在什么地方栽跟头

```
用户问「OA 系统怎么登录」
文档标题《OA 办公系统使用指南》

向量检索：「OA」是低频专名，embedding 表示不准，
          「登录」语义又太通用 → 可能召回到一堆「登录」相关但非 OA 的文档

BM25：直接匹配「OA」「登录」字面词 → 精准命中 OA 指南
```

规律：**越是专有名词、产品代号、缩写、数字编号，向量越不可靠，BM25 越有优势**。企业文档里这类词极多（系统名、流程名、表单号），所以 BM25 在企业知识库不是可选项。

---

## 2. BM25 是什么（一句话）

BM25 是经典的关键词检索打分算法：一个 chunk 的得分，取决于**查询词在它里面出现的频率**（词频）和**这些词有多稀有**（逆文档频率）。稀有词命中加分多，常见词（「的」「公司」）加分少。它不懂语义，只看字面，但字面匹配恰恰是向量的盲区。

---

## 3. 两种实现方式

| 方式 | 怎么做 | 适合 |
|------|--------|------|
| Postgres 全文检索 | 用 `tsvector` + `ts_rank`（[05-ingest/05](../05-ingest/05-write-pgvector.md) 已存 tsv） | 和向量同库，简单 |
| 应用层 `rank-bm25` | Python 库，自己算 | 中文分词更可控 |

中文场景下，Postgres 内置分词不行，要么上 `zhparser` 扩展，要么用应用层方案。这里给应用层 `rank-bm25`（配合中文分词）的实现，更透明：

```python
# retrieve/bm25.py
import jieba
from rank_bm25 import BM25Okapi
from db import get_conn

def tokenize(text: str) -> list[str]:
    return [w for w in jieba.lcut(text) if w.strip()]

# 启动时构建索引（chunk 不多，内存放得下）
def build_bm25():
    rows = get_conn().execute(
        "SELECT id, doc_id, content FROM chunks "
        "JOIN documents USING... WHERE status='active'"
    ).fetchall()
    corpus = [tokenize(r["content"]) for r in rows]
    return BM25Okapi(corpus), rows

bm25, chunk_rows = build_bm25()

def bm25_search(question: str, k: int = 20) -> list[dict]:
    scores = bm25.get_scores(tokenize(question))
    ranked = sorted(zip(scores, chunk_rows), key=lambda x: -x[0])[:k]
    return [{**r, "bm25_score": s} for s, r in ranked]
```

> 注：几千 chunk 内存建索引完全 OK；文档更新后重建即可。规模再大时改用 Postgres/Elasticsearch 的全文索引。

## 5. 中文分词是 BM25 的关键

BM25 的效果**强依赖分词质量**。中文不像英文有空格，「报销额度」要切成 `["报销", "额度"]` 而非 `["报", "销", "额", "度"]`：

```python
import jieba
# 给业务专名加自定义词典，避免被切碎
jieba.add_word("OA系统")
jieba.add_word("差旅报销")
```

把企业专有名词加进分词词典，BM25 才能把它们当一个词精确匹配。这是中文 BM25 容易被忽略但收益很大的一步——专名切碎了，BM25 的最大优势（精确匹专名）就没了。

## 6. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 只用向量，不用 BM25 | 专名/缩写召回差 | 两路都用 |
| 中文不分词直接 BM25 | 切成单字，匹配乱 | jieba 等中文分词 |
| 专名没加词典 | 被切碎，匹不中 | 自定义词典加专名 |
| 加了 BM25 不单独评估就融合 | 不了解它的脾气 | 先单测看互补性 |
| 大规模还用内存 BM25 | 内存/重建吃力 | 上全文索引引擎 |
