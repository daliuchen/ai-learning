# EKB 48：成本与延迟预算——给每一步算账

> **一句话**：上线前要给「单次问答」算两笔账——花多少钱、花多长时间。把成本和延迟拆到每个环节（embed/检索/rerank/改写/生成），就能看出钱和时间花在哪、哪里能省。本篇给出预算模型和优化手段。

---

## 1. 单次问答的延迟拆解

把 Query 流的每一步耗时列出来（示意值）：

```
query 改写（小模型）   ~300ms   ← 一次额外模型调用
embed query           ~30ms
向量检索（pgvector）   ~20ms
BM25 检索             ~15ms
rerank（20候选）       ~150ms   ← cross-encoder
生成（流式，到首字）    ~500ms   ← TTFT，用户感知的「快慢」
─────────────────────────────
首字延迟（TTFT）       ~1秒
```

关键认知：**用户感知的是「首字延迟（TTFT）」，不是总时长**。流式下，答案开始蹦字用户就觉得快了。所以优化重点是**让第一个字尽早出现**——改写和 rerank 都堵在生成之前，是 TTFT 的大头。

---

## 2. 延迟优化手段

| 手段 | 省哪里 | 代价 |
|------|--------|------|
| 改写用小模型/可跳过 | 省 ~300ms | 简单问题跳过改写 |
| rerank 候选数调小 | 省 rerank 时间 | 召回窗口变窄 |
| 并行化检索 | 向量/BM25 并行而非串行 | 实现复杂度 |
| 缓存高频问题的答案 | 省整条链 | 答案时效（文档更新要失效）|
| 流式 | 改善 TTFT 体感 | 已默认 |

最划算的两个：**简单问题跳过改写**（省 300ms）、**热门问题答案缓存**（命中直接返回）。后者要注意文档更新时让缓存失效。

---

## 3. 单次问答的成本拆解

按 token 算钱（示意，价格随模型变）：

```
query 改写（小模型）  输入~50 + 输出~50 token    ≈ 便宜
生成（强模型）        输入 = 系统提示 + 4个片段 ≈ 2000 token
                     输出 ≈ 200 token
─────────────────────────────────────────
主要成本在「生成」的输入 token（片段占大头）
```

成本的大头是**生成环节的输入 token**，而输入里**检索片段**占最多。所以省钱的方向：

- **少塞片段**：rerank 后只取真正相关的 top 3-4，别贪多（噪声还伤准确率）
- **缓存系统提示**：prompt caching（上一篇）
- **改写/兜底用小模型**：非核心环节不用强模型
- **片段去重/裁剪**：重叠内容、超长片段裁掉（[07 手册裁剪](/docs/07-context-engineering/03-retrieval/04-rank-trim)）

---

## 4. 强模型 vs 小模型的分工

不是所有环节都要强模型，按重要性分配：

| 环节 | 模型档位 | 理由 |
|------|----------|------|
| 生成答案 | 强（Claude/GPT-4o） | 「忠于文档、不编」需要好的指令遵循 |
| query 改写 | 小（mini） | 改写简单，小模型够 |
| 兜底判断 | 随生成 | 已在生成里 |
| 评估 judge | 中~强 | 判断要准 |

**把钱花在刀刃上**——生成是质量命门，用强模型；改写是辅助，用小模型。一刀切全用强模型，成本翻几倍但收益有限。

---

## 5. 设定预算目标并监控

预算不是算完就忘，要设目标 + 持续监控：

```
目标（示例）：
  TTFT < 1.5s（P95）
  单次问答成本 < $0.01
  总延迟 < 4s（P95）

监控（下章可观测）：
  每次问答记录各环节耗时 + token 用量 → 看 P50/P95
  超标告警
```

预算超标时，回到上面的拆解表，看是哪个环节涨了——是 rerank 候选给多了？还是片段塞多了？**有拆解才有的放矢。**

---

## 6. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 只看总延迟 | 忽略了 TTFT 才是体感 | 优化首字延迟 |
| 所有环节用强模型 | 成本翻倍 | 按重要性分配模型档 |
| 贪多塞片段 | 贵且噪声伤准确率 | rerank 后取精 |
| 不设预算目标 | 上线后成本失控 | 设目标 + 监控 |
| 答案缓存不失效 | 文档更新后答旧内容 | 缓存绑文档版本 |
