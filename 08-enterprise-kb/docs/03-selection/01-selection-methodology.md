# EKB 11：选型方法论——先定边界，再选技术

> **一句话**：技术选型最常见的错误，是**一上来就比较 Qdrant vs Milvus**。正确的顺序是反的——先回答「规模多大、内容什么形态、谁在用、多久更新」这几个边界问题，答案自然会把选项砍到只剩一两个。选型不是选「最强」，是选「最匹配」。

---

## 1. 错误的选型 vs 正确的选型

```
❌ 错误：listen 到「做知识库」→ 立刻 Google「最好的向量数据库 2026」
        → 看到 Milvus 性能榜第一 → 选 Milvus
        → 几千篇文档用上了能扛十亿向量的库，运维成本爆炸

✅ 正确：先问「我有多少文档？切完多少 chunk？」→ 几千 chunk
        → 这个规模 pgvector 完全够 → 还能少维护一个组件
        → 选 pgvector
```

差别在于：**正确的选型是「需求倒推」，错误的选型是「参数崇拜」。**

---

## 2. 选型前必答的 5 个边界问题

任何 RAG 系统选型，先回答这 5 个，答案决定一切：

| 问题 | 影响什么 | 我们的答案 |
|------|----------|-----------|
| **规模**：多少文档？切完多少 chunk？ | 要不要重型向量库 | 几百~几千篇 → 几千~几万 chunk |
| **内容形态**：FAQ？长文档？混合？ | 分块策略 | 结构化长文档（制度/手册） |
| **谁在用**：内部？外部？多少并发？ | 权限、并发、SLA | 内部员工，低并发 |
| **更新频率**：静态？天天改？ | 索引/增量策略 | 偶尔更新（制度改版） |
| **只问答还是要干活**：问答？agentic？ | 要不要 Agent 编排 | 纯问答 |

把这张表填完，你会发现 80% 的「该用什么」答案已经浮现了。

---

## 3. 从边界答案推出技术选择

逐条看我们的答案如何砍掉选项：

### 规模「几千~几万 chunk」→ pgvector

这个量级，向量检索全表扫都不慢，加个 HNSW 索引更是毫秒级。重型向量库（Milvus/Qdrant 集群）解决的是「亿级向量 + 独立扩缩容」，我们用不上。pgvector 还能和业务数据同库，方便 ACL join。

### 内容「结构化长文档」→ 语义分块

制度手册有清晰的标题层级。固定长度硬切会切碎小节，应该**按结构切**，每片带 section 元数据。

### 用户「内部、低并发」→ 不需要重型基建

不用考虑 CDN、限流、多区域。一台机器 + 一个 Postgres 足够。精力省下来花在检索质量上。

### 更新「偶尔」→ 增量重跑即可

不需要实时索引管道。文档变了，重新 ingest 那一篇就行。

### 「纯问答」→ 不需要 Agent 编排

没有「查订单、调 API」这类动作。一条固定的「检索→生成」pipeline 就够，不需要 LangGraph/Agent 那种动态编排。

## 4. 选型的两个反直觉原则

### 原则一：少一个组件就少一份运维

每引入一个新组件（向量库、缓存、消息队列），都是一份要部署、监控、排障、升级的负担。**能用现有组件（Postgres）解决的，就别引新的。** 这对小团队/教学项目尤其重要。

### 原则二：可替换 > 一步到位

选型不确定时，把那一层做成**可替换的接口**，而不是赌一个「最优解」。比如 embedding 模型——中文场景哪个最好需要实测，那就把它抽象成一个函数，随时能换。详见下一篇和 [03-embedding-selection](./03-embedding-selection.md)。

## 5. 一个选型决策的完整推导示例

「我要不要上 Redis 做缓存？」

```
问：当前瓶颈是什么？ → 还没上线，没有瓶颈
问：有重复查询吗？  → 有，高频问题会重复
问：Postgres 扛得住吗？ → 低并发，完全扛得住
结论：现在不上 Redis。先用 prompt caching（模型层）降本（10章），
      真出现 DB 压力再说。← 不为「将来可能」过度工程
```

每个「要不要上 X」都这样推一遍，你的架构会自然保持精简。

---

## 6. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 先比技术再想需求 | 选出过度工程的方案 | 先答 5 个边界问题 |
| 追性能榜单 | 用上了用不到的能力 | 选匹配规模的，不选最强的 |
| 为「将来可能」上组件 | 运维负担、复杂度 | 出现真瓶颈再上 |
| 赌一个「最优」选型 | 错了要推翻 | 不确定的层做成可替换 |
