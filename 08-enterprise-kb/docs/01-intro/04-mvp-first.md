# EKB 04：MVP 优先——先让它能跑，再让它变好

> **一句话**：不要一上来就追求「混合检索 + rerank + 权限 + 流式 UI」全套。先用**最朴素的单路向量检索**打通端到端，拿到一个能问能答的 baseline，再用评估指标驱动一项项加增强。能跑的 baseline，比纸上完美的架构有用一百倍。

---

## 1. 为什么先做 MVP

新手做 RAG 项目最常见的死法：**还没跑通就想把所有增强一次性堆上去**，结果改一处坏一处，根本不知道哪个环节出了问题。

正确的节奏是**纵切**而非横切：

```
❌ 横切：先把「分块」做到完美，再把「检索」做到完美，再…… → 很久看不到端到端效果
✅ 纵切：先用最简单的方式打通「问 → 答」整条链 → 再逐段增强
```

MVP 的价值：
- **暴露真问题**：很多坑只有端到端跑起来才会出现（数据格式、prompt 拼接、延迟）
- **建立 baseline**：有了能打分的起点，后面每个增强才能量化「涨了多少」
- **保持动力**：一个能用的东西，比一堆完美的零件更让人想继续

---

## 2. 这个项目的 MVP 砍到哪

我们的完整系统有这些能力。MVP 阶段**只留最细的一条线**：

| 能力 | 完整版 | MVP |
|------|--------|-----|
| 分块 | 语义分块 + 元数据 | 固定长度切，先能切就行 |
| 检索 | 混合 + rerank + query 改写 | **单路向量检索，top-k** |
| 生成 | 结构化输出 + 引用 + 兜底 | 先能带引用就行 |
| 权限 | 按角色过滤 | **先不做，全部可见** |
| 前端 | 流式 + 引用卡片 + 后台 | **先用命令行/脚本调** |
| 评估 | 完整测试集 | 一个小测试集（这个**不能砍**） |

注意：**评估不能砍**。它是标尺，砍了就是闭眼改代码。其它都可以从最简开始。

## 3. MVP 的最小数据流

```python
# MVP 版检索 + 生成（伪代码，完整实现见 06 章）
def answer(question: str) -> Answer:
    # 1. 把问题 embed
    q_vec = embed(question)
    # 2. 单路向量检索 top-5（先不管权限、不管 BM25）
    chunks = db.query(
        "SELECT content, doc_id FROM chunks "
        "ORDER BY embedding <=> %s LIMIT 5",
        [q_vec],
    )
    # 3. 拼 prompt，让模型带引用回答
    return generate(question, chunks)
```

就这么朴素。能跑通这 20 行，你就有了一个可以打分、可以迭代的真实系统。

## 4. 然后按指标驱动地加增强

有了 MVP 和测试集，后续每一步都遵循同一个循环（这正是 04/07 手册讲的「评测驱动迭代」）：

```
测当前指标 → 加一个增强 → 重测 → 涨了就留，没涨/变差就回退
```

我们会一项一项加，每加一项都用第 04 章的指标量化：

| 增强 | 预期改善 | 在哪一章 |
|------|----------|----------|
| 语义分块（替代固定切） | 召回更完整 | 05 章 |
| BM25 + 混合检索 | 专有名词/缩写召回 | 07 章 |
| Rerank | 精排准确率 | 07 章 |
| Query 改写 | 口语问题召回 | 07 章 |
| 权限过滤 | 安全性（不是准确率） | 08 章 |

**关键**：每次只加一项，单独测量。一次加三项，涨了你也不知道是哪项的功劳，跌了更不知道怪谁。这是 [04-prompt-engineering 的「一次只改一处」](/docs/04-prompt-engineering/02-process/04-iteration-loop) 在 RAG 上的应用。

---

## 5. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 一次性堆所有增强 | 跑不通、不知道哪坏了 | 先 MVP 纵切打通 |
| 连评估也省了 | 闭眼改，凭感觉 | 评估是 MVP 唯一不能砍的 |
| 一次改多项再测 | 涨跌都归因不了 | 一次只加一项，单独测 |
| MVP 阶段就纠结分块细节 | 迟迟看不到端到端 | 先固定切，能跑就行，05 章再优化 |
| baseline 没存档 | 后面无法对比 | 记录 MVP 的指标作为起点 |
