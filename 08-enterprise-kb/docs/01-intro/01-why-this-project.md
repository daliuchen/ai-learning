# EKB 01：为什么是这个项目——读完手册 ≠ 会落地

> **一句话**：前 7 本手册教你「每个零件是什么、怎么用」，但「会用零件」和「能交付一个系统」之间隔着三道坎——选型、拼装、生产现实。这本手册用一个**企业知识库**项目，把这三道坎一次性补上。

---

## 1. 你已经会的，和你还不会的

假设你认真读完了前 7 本：你知道 embedding 怎么选、chunk 怎么切、rerank 怎么用、上下文怎么管、Pydantic AI 怎么写 Agent。这些都是**零件知识**。

但现在老板说「给公司做个内部知识库问答」，你大概率会卡在这些地方：

| 卡点 | 手册讲过吗 | 真实问题 |
|------|-----------|----------|
| 该上向量库还是直接塞长上下文？ | 各讲了，但没讲怎么**选** | 几千篇文档，到底要不要 Qdrant？ |
| LangChain 还是 Pydantic AI 还是裸 SDK？ | 各讲了 | 这个场景哪个最省心？ |
| chunk + embed + 检索 + 生成怎么粘成一条线？ | 零件都有 | 中间的胶水、数据流没人教 |
| 检索回来一堆不相关的怎么办？ | 提了 rerank | demo 能跑，真数据上准确率惨不忍睹 |
| 谁能看哪些文档？ | 几乎没讲 | HR 文档不能给全员看，这是企业刚需 |

**这本手册补的就是右边这一列。**

---

## 2. 三道坎，逐个说清

### 坎一：判断力（选型）

真实需求来了，方案不是「用最酷的技术」，而是「用最合适的」。
- 几千篇文档 → 其实**不需要**重型向量库，pgvector 足够
- 相对定型的问答 pipeline → 上 LangChain 反而是负担
- 中文为主 → embedding 默认 OpenAI 可能不是最优

这些判断，手册给了零件参数，但「面对具体场景怎么拍板」要靠方法论。本手册第 03 章专门讲。

### 坎二：拼装能力（胶水）

单个组件会用，但把它们粘成一条能跑的数据流，中间全是没人写的细节：
- chunk 切完，元数据（属于哪篇文档、哪个小节）怎么带？
- 检索回来的片段，怎么塞进 prompt 才能让模型正确引用？
- 流式回答 + 结构化引用，前后端怎么配合？

本手册第 05-06 章把这条线一节一节焊起来。

### 坎三：生产现实（会炸在哪）

教程永远停在「it works!」那一刻。但真实系统的功夫全在那之后：
- 检索回来的是垃圾 → 准确率上不去
- 模型编答案 → 企业场景里这是**事故**
- 答不出却硬答 → 比不答更糟
- 成本爆了、延迟高了 → 没法上线

本手册第 04（评估先行）、07（检索增强）、10（生产化）章，专门趟这些坑。

## 3. 为什么选「企业知识库」当载体

在所有 AI 应用里，企业知识库（内部帮助中心）是**最经典、最值得做、又最能暴露真实问题**的 RAG 场景：

- **需求真实**：几乎每家公司都需要，员工查制度/流程/产品文档
- **零件全用上**：embedding、分块、检索、rerank、上下文、结构化输出，一个不落
- **有独特难点**：权限（谁能看哪些文档）——这是它区别于「玩具 RAG demo」的核心，也是教程从不讲的
- **可量化**：答得准不准、引用对不对，都能用指标衡量，不靠「感觉还行」

一句话：**它小到能在一个项目里讲透，又真到能暴露所有生产问题。**

---

## 4. 这本手册不教什么（先划清边界）

为了讲透，我们刻意做减法：

- ❌ **不做对外客服系统**——没有访客、坐席、转人工、会话状态机。那是另一套重心（重在交接逻辑），会冲淡 RAG 主线。本手册聚焦**对内问答**。
- ❌ **不追求生产级健壮**——这是教学项目，目标是「讲清楚每个决策」，不是「扛住百万 QPS」。
- ❌ **不堆砌框架**——能自己写薄薄一层讲清楚的，绝不引重型框架来制造黑盒。

边界详见 [02-design/03-boundary-nongoals](../02-design/03-boundary-nongoals.md)。

## 5. 常见误区

| 误区 | 现实 |
|------|------|
| 「读完手册就能做项目了」 | 零件 ≠ 系统，选型/拼装/生产是另外三本事 |
| 「先把功能堆全再说」 | 不评估就堆功能 = 不知道在变好还是变坏 |
| 「RAG demo 跑通 = 能上线」 | demo 用干净数据，真数据上检索质量是分水岭 |
| 「权限最后加就行」 | 权限要在**检索阶段**设计，事后补会推翻数据模型 |
| 「用最新最酷的技术准没错」 | 几千篇文档上重型向量库 = 过度工程 |
