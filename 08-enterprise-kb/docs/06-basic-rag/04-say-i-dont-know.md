# EKB 30：答不出就承认——企业知识库的底线

> **一句话**：知识库里没有的，**绝不能编**。「公司明年涨薪吗」这种没文档支撑的问题，必须老实说「没找到」。编一个看似合理的答案，在企业场景是事故。本篇讲怎么用「检索阈值 + 生成约束 + 后处理」三道防线守住这条底线。

---

## 1. 为什么这是底线，不是优化项

普通聊天机器人编一句无伤大雅，但企业知识库不同：

```
员工问「试用期能休年假吗」
知识库里没有这条规定
❌ 模型编：「试用期可以休年假」→ 员工照做 → 违规 → 出事追责到系统
✅ 模型答：「未找到相关规定，请咨询 HR」→ 安全
```

**编答案的代价是真实的业务损失和信任崩塌。** 所以「答不出就承认」的优先级高于「答得多」「答得全」。评估里兜底正确率权重最高（[04-eval/03](../04-eval/03-metrics.md)），就是这个原因。

---

## 2. 三道防线

「不编」不能只靠一处，要层层设防：

```
防线 1（检索层）：相关度太低 → 根本不给模型片段
防线 2（生成层）：系统提示强制「片段不足就 found=false」
防线 3（后处理）：声称 found 但无合法引用 → 降级为未找到
```

任何一道都可能漏，三道叠加才稳。

---

## 3. 防线一：检索阈值

如果检索回来的片段相关度都很低，说明知识库里没有——**直接不进生成**：

```python
RELEVANCE_THRESHOLD = 0.35   # 评估集校准

def retrieve_relevant(question, k=5):
    results = vector_search(question, k)
    relevant = [r for r in results if r["distance"] < RELEVANCE_THRESHOLD]
    return relevant            # 可能返回空列表
```

空列表 → `generate` 直接返回兜底（上一章已实现），连模型都不调。这道防线最干脆——没有片段，模型就没有「编」的素材。

---

## 4. 防线二：生成层约束

即使有片段，片段也可能**沾边但不足以回答**。这时靠系统提示让模型自己判断：

```python
system_prompt = (
    "只能基于【文档片段】回答。\n"
    "如果片段提到了相关话题但不足以给出确切答案，"
    "不要猜测或补全，把 found 设为 false。\n"
    "宁可回答「未找到」，也不要给可能错误的答案。"
)
```

关键措辞：**「宁可漏，不可错」**要明确传达给模型。很多模型默认倾向「尽力给个答案」，必须显式压制这个倾向。这是 [04 手册](/docs/04-prompt-engineering/03-techniques/01-instructions) 讲的「对抗模型默认行为」。

---

## 5. 防线三：后处理降级

模型有时会「嘴上说 found=true，但其实没真正引用任何片段」。后处理兜住：

```python
def enforce_grounding(answer: Answer, chunks: list[dict]) -> Answer:
    valid = {c["doc_id"] for c in chunks}
    answer.cited_doc_ids = [i for i in answer.cited_doc_ids if i in valid]
    # found 但无合法引用 = 没真正基于文档 → 降级
    if answer.found and not answer.cited_doc_ids:
        answer.found = False
        answer.text = "未在知识库中找到相关信息，建议联系对应部门确认。"
    return answer
```

规则：**有依据才算找到**。没有任何合法引用却声称找到，一律当没找到处理。

---

## 6. 兜底话术也要好好设计

「没找到」不是冷冰冰地拒绝，而是**有帮助地兜底**：

```
❌ 差：「未找到。」
✅ 好：「未在知识库中找到关于『试用期年假』的规定，
        建议咨询 HR（hr@company.com）或查看《考勤制度》原文。」
```

好的兜底话术：承认没找到 + 给下一步建议（找谁/看哪）。可以根据问题的 `space` 推荐对应部门联系方式。这让「答不出」也是一次有用的交互。

---

## 7. 常见坑

| 坑 | 后果 | 正确做法 |
|----|------|----------|
| 只靠 prompt 防编造 | 模型偶尔还是编 | 三道防线叠加 |
| 没有检索阈值 | 无关片段也喂给模型 | 阈值过滤 |
| 不压制模型「尽力回答」倾向 | 沾边就硬答 | 显式「宁可漏不可错」 |
| found 不校验引用 | 空口说找到 | 无引用则降级 |
| 兜底话术生硬 | 体验差 | 承认 + 给下一步建议 |
